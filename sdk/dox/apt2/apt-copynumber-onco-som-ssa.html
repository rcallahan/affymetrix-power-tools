Introduction

apt-copynumber-onco-som-ssa implements the somatic mutation analysis pipeline for OncoScan arrays. apt-copynumber-onco-som-ssa is designed to be run sequentially with apt-copynumber-onco-ssa. Input for each OncoScan experiment consists of a pair of CEL files for AT and GC channels and an OSCHP file generated by apt-copynumber-onco-ssa. Rather than generating a separate output file for somatic mutation results, apt-copynumber-onco-som-ssa will add a somatic mutation data group to an existing OSCHP file. apt-copynumber-onco-som-ssa supports two modes of operation: single sample mode and "with matched normal" analysis. In the "with matched normal" mode a pair of cel files is identified as “normal” and a pair as “tumor” and the normal data is used to identify heterozygous markers for use by the algorithm when analyzing the tumor pair. In this analysis the log2ratios are still calculated relative to the reference model file. (This is not a “paired analysis” where the tumor is compared to the normal to calculate the log2 ratio data.) experiments (i.e. four CEL files) are consisting of a “normal” and a “tumor” sample are submitted.

Quick Start

Input/Output Files

The following input files are required by apt-copynumber-onco-som-ssa:

Name File extension/type Description
reference-file REF_MODEL (HDF5) Reference data generated by apt-copynumber-onco-ref
som-thresholds-file text Thresholds for Somatic Mutation calls
annotation-file annot.db (SQLite) Annotation data for probes and probesets
x-probes-file chrXprobes (text) List of X-chromosome probes
y-probes-file chrYprobes (text) List of Y-chromosome probes
cel-pairs-file tsv Cel file pairs (AT and GC channel for each sample) for experiments

The following output files are updated by apt-copynumber-onco:

Name File extension/type Description
OSCHP file OSCHP (HDF5) Copynumber and other analysis data for each experiment

CEL files lists:

The CEL file pairs are specified in a tab delimited text file. In single-sample mode this file will have either two or three columns, with two columns representing the AT-channel and the GC-channel CEL files. In normal/tumor pair mode, the file will have either four or five columns, with four columns representing the normal AT and GC-channel CEL files, and the tumor AT and GC-channel CEL files. The optional third or fifth column can be used to specify the name of the output OSCHP files. (If omitted an OSCHP file name is generated based on the CEL file names.)

The first row of the cel-pairs-file should contain the column names. Supported column names are:

        atchannelcel    - the AT channel CEL file
        gcchannelcel    - the GC channel CEL file
        atchannelmatchednormalcel       - the AT channel CEL file for matched normal sample
        gcchannelmatchednormalcel       - the GC channel CEL file for matched normal sample
        result          - the OSCHP file name to be generated (optional)

See OncoScan single-sample analysis for examples.
