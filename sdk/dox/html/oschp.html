Introduction

OSCHP files contain analysis results generated by the APT OncoScan analysis pipeline and tools or by the OncoScan Console software. This document is intended to accompany one or more sample OSCHP files and to provide guidance for interpreting the data in the files. It is recommend that an HDF5 viewing tool, such as HDFView (http://www.hdfgroup.org/hdf-java-html/hdfview/index.html) be used to inspect the contents of the files.

Format

The format of the OSCHP files is the HDF5 binary format. Data is organized into a number of groups with each group containing zero or more sub-groups and zero or more datasets. Each sub-group may, in turn, contain zero or more sub-sub-groups and/or datasets, yielding a nested tree structure. Each dataset may have one or more columns or fields, and multiple rows. Consequently every dataset group or dataset can be described by a unique path in the file, such as /Group1/Group2/Dataset.

Full details of all the features of the HDF5 format are outside the scope of this document (see http://www.hdfgroup.org/HDF5/doc/index.html for more details).

The tree structure of the HDF5 files is as follows (datasets are in bold):

  • Dset_IO_HDF5_Gdh (General Header Data)
    • _&keyvals (data source: affymetrix-multi-data-type-analysis)
    • Dset_IO_HDF5_Gdh:0 (AT channel array)
      • _&keyvals (data source: affymetrix-calvin-intensity)
      • Dset_IO_HDF5_Gdh:0:0
        • _&keyvals (datasource: affymetrix-calvin-scan-acquisition)
        • Dset_IO_HDF5_Gdh:0:0:0
          • _&keyvals (datasource: affymetrix-calvin-array)
    • Dset_IO_HDF5_Gdh:1 (GC channel array)
      • _&keyvals (data source: affymetrix-calvin-intensity)
      • Dset_IO_HDF5_Gdh:1:0
        • _&keyvals (datasource: affymetrix-calvin-scan-acquisition)
        • Dset_IO_HDF5_Gdh:1:0:0
          • _&keyvals (datasource: affymetrix-calvin-array)
  • AlgorithmData (Data produced during algorithm development)
    • MarkerABSignal (internal use only)
  • Chromosomes (Summary analysis results for each chromosome)
    • Summary
  • Genotyping (Genotyping analysis results from BRLMM-P)
    • Calls
    • SigSnpReport
  • ProbeSets (Analysis result for each probeset)
    • AllelicData
    • CopyNumber
  • Segments (Analysis results for segments)
    • TuscanCN
    • TuscanUnmergedCN (internal use only)
    • LOH
  • SomaticMutation
    • SomCall

Note that the structure is an extension of the existing structure used in Affymetrix CYCHP and CNCHP file formats.

Header Data

The Dset_IO_HDF5_Gdh group contains header data in the form of key/value pairs. The groups structure is based on the General Data Header, or GDH, section of the Affymetrix Command Console binary format. The group contains a series of nested subgroups named Dset_IO_HDF5_Gdh:<level1>:…:<levelN> each of which may contain a dataset named _&keyvals.

Specifically, for a pair of OncoScan arrays, the group Dset_IO_HDF5_Gdh will contain two sub-groups Dset_IO_HDF5_Gdh:0 and Dset_IO_HDF5_Gdh:1, and a dataset _&keyvals. The sub-groups /Dset_IO_HDF5_Gdh/Dset_IO_HDF5_Gdh:0 and /Dset_IO_HDF5_Gdh/Dset_IO_HDF5_Gdh:1 will contain GDH data copied from the CEL files for the AT and the GC arrays respectively, including data generated by Affymetrix Command Console and upstream analysis steps (see .CEL file documentation for details), while the dataset /Dset_IO_HDF5_Gdh/_&keyvals will contain summary and metadata generated by the APT OncoScan pipeline. For a tumor experiment as part of a matched normal/tumor pair (i.e. set of four CEL files), the group Dset_IO_HDF5_Gdh will contain four subgroups, corresponding to the GDH data for all four CEL files.

Each row of a _&keyvals dataset contains data about one key/value pair. The datasets contain three fields: key (the key of the pair), val (a string representing the value of the pair), and type (a string representing the type of the value, e.g. “string8” for a string of 8-bit characters, “int32” for a 32-bit integer, or “float32” for a 32-bit float).

Each key/value pair contains summary information or details of the program version and analysis parameters. The first key/value pair of each _&keyvals dataset will have the key “data-source” and indicate the origin of the GDH group (this is used in mapping back to the Affymetrix Command Console binary format). The row with key “data-source” in the /Dset_IO_HDF5_Gdh/_&keyvals will have the value “affymetrix-multi-data-type-analysis”.

Summary statistics begin with the prefix affymetrix-chipsummary- while analysis parameters begin with the prefix affymetrix-algorithm-param-.

Chromosome Data Group

The Chromosomes data group consists of a single data set called Summary. The Summary data set contains summary information for each chromosome. This data set will contain the following columns:

Column Name Column Type Description
Chromosome UByte The values in this column are defined as 1-22, 24 (for X), 25 (for Y) and 26 (for MT) and 255 (for no value). The chromosome display values (1-22, X, Y, MT) are stored in the Display field.
Display Text label for the chromosome.
StartIndex UInt Zero-based reference to the row in the CopyNumber data set of the ProbeSets data group containing results for this chromosome
MarkerCount UInt The number of markers in the chromosome
MinSignal Float Minimum log2 ratio value found in the chromosome*
MaxSignal Float Maximum log2 ratio value found in the chromosome*
MedianCnState Float Median copynumber state reported by TuScan
HomFrequency Float Frequency of Homozygosity
HetFrequency Float Frequency of Heterozygosity
LOH Float Proportion of markers reporting LOH calls per chromosome
MedianSignal Float Median log2 ratio value found in the chromosome*

The log2 ratios are trimmed log2 ratios by default, but the values might go beyond the trimmed values due to the GC correction and/or wave correction steps.

ProbeSets Data Group

The Probesets data group consists of two data sets, called AllelicData and CopyNumber containing analysis results specific to the probesets.

The data rows for each chromosome are consecutive, in a single block, starting at the index given in the "Chromosomes/Summary" data set in the StartIndex column. The number of data rows for each chromosome is exactly equal to the value given in the "Chromosomes/Summary" data set in the MarkerCount column.

The CopyNumber data set will contain the following columns:

Column Name Column Type Description
ProbeSetName Ascii The name of the probeset. The probeset names begin with "C" (copy number probes) or "S" (SNP probes).
Chromosome UByte The values in this column are defined as 1-22, 24 (for X), 25 (for Y) and 26 (for MT) and 255 (for no value). (Corresponding chromosome display values (1-22, X, Y, MT) are stored in /Chromosomes/Summary dataset).
Position UInt Genomic position in the chromosome. The Position values for each chromosome are strictly increasing. Each position can be assosciated with one probe only.
Log2Ratio Float Log2 ratio (signal / median of Reference signal)
WeightedLog2Ratio Float Signal restored log2 ratio value
SmoothSignal Float Each value is the result of a kernel smooth over a window of genomically contiguous log2 ratios. By default this is converted to a copy number value using a log-linear model that fits each CN state to its expected log2 ratio. This is useful for seeing mosaics. (Internal use only)
NormalDiploid UInt 1 = diploid, 0 = non-diploid (Internal use only)

The AllelicData data set houses data summarizing allele patterns and contains the following columns:

Column Name Column Type Description
ProbesetName Ascii The name of the probeset
Chromosome UByte Chromosome number, as for CopyNumber data set
Position UInt Genomic position in the chromosome
AllelicDifference Float Allelic difference value
BAF Float B-allele frequency (range 0.0-1.0)

Segments Data Group

The Segments data group contains data sets housing segments data. The Segments group currently contains the following datasets:

  • TuscanCN – The merged segments reported by the TuScan algorithm.
  • LOH - Segments reported by the Loss of Heterozygosity algorithm.
  • TuScanUnmergedCN – This table is for internal use only.

Both the TuscanCN and the LOH-Segments data sets in this data group will contain the following columns:

Column Name Column Type Description
SegmentID UInt Segment identifier unique to the OSCHP file
Chromosome UByte The chromosome display values (1-22, X, Y, MT) are stored in the data set header. The values in this column are defined as 1-22, 24 (for X), 25 (for Y) and 26 (for MT) and 255 (for no value)
StartPosition UInt Start genomic position of the segment. The StartPosition column is strictly increasing within each dataset.
StopPosition UInt Ending genomic position of the segment. The StopPosition is always greater than the StartPosition in the same row and less than the StartPosition in the next row.
MarkerCount Int Number of markers in the segment.

In addition to the columns above, TuscanCN data set contains the following columns:

Column Name Column Type Description
TotalCN Float nA + int nB or fraction in AlternateMode
BinnedCN String The representative value for the bin into which the CN value falls. (If the representative value is “as-is” then the actual value, rounded to a given number of decimal places, is shown).
MedianLog2ratio Float Median log 2 ratio
MedianBAF Float Median of markers identified as heterozygous, after mirroring any marker BAFs above 0.5 to the equivalent value below 0.5. It is NaN if the number of heterozygous markers in the segment is below 10 or the percent of homozygous markers is above 85%

The LOH data set contains the following additional columns:

Column Name Column Type Description
MeanMarkerDistance UInt Mean distance between markers in the segment
LOH UInt The value of this column is 1 when a Loss of Heterozygosity is found, 0 when not found.
Confidence Float The ratio of the probability of the SCAR measurements under the LOH model to the sum of the probability under each of the LOH and non-LOH models.

Genotyping Data Group

The Genotyping data group contains two data set, Calls, capturing data from the BRLMM-P genotyping algorithm, and SigSnpReport, capturing the results of the signature SNP analysis of the AT and CG channels.

The Calls data set will contain the following columns:

Column Name Column Type Description
Index UInt This column contains the index relative to the probeset in the CopyNumber data set of the ProbeSet data group
ProbeSetName Ascii The name of the probeset.
Call UInt Genotype calls (any ForcedCall that does not meet a given genotype confidence threshold will result in no call)
Confidence Float Genotype confidence
ForcedCall UInt Genotype calls without restriction to any genotype confidence
ASignal Float Summarized signal for A allele
BSignal Float Summarized signal for B allele
SignalStrength Float 0.5* (log2 A signal + log2 B signal)
Contrast Float log2 B signal – log2 A signal

The SigSnpReport data set contains data from the genotyping results of a set of signature SNPs which are used in checking that the AT and GC channel CEL files are correctly matched and classified. (Note the fields of this dataset follow the format of the Signature SNP report file.) The dataset will contain the following columns:

Column Name Column Type Description
Filename String The OSCHP file for the experiment
CEL filename String The name of one of the CEL files in the CEL file pair for the experiment
Channel String The channel for the CEL file: either “AT” or “GC”
CelPairCheckStatus String The result of the checking for the signature snps: either “Pass” or a string indicating the likely source of problems.
CelPairCheckCallRate Float The percent of signature SNPs for which calls were generated
CelPairCheckCompareRate Float The percent of signature SNPs used in comparing the two CEL files
CelPairConcordance Float The concordance for the signature SNPs across the two CEL files.
SIG_nnn (where nnn is an integer) String For signature SNP nnn, the genotyping call. Either “AA”, “AB”, “BB” or “NN”.
